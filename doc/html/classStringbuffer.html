Detailed Description

Buffer containing text to output.

Stringbuffer implements a buffer gathering characters that have to be outputed on the screen or a differenct output device. If the buffer is full or the user wishes to print the content of the buffer on the output device method flush() is called. To enable Stringbuffer to work with different output devices method flush() has to be implemented by the subclasses of Stringbuffer. Since every subclass needs access to the variables and methods of the Stringbuffer every variable and method of Stringbuffer is declared 'protected'.

virtual void Stringbuffer::flush ( ) [protected, pure virtual]

flush the content of the buffer to the screen

Method flush() prints the curent content of the buffer. It is called automaticaly as soon as the buffer is full or manualy if an output is forced. After printing the buffer content the buffer pointer is reseted. To enalbe the Stringbuffer to work with different output mechanisms the method has to be implemented in a subclass of Stringbuffer.
